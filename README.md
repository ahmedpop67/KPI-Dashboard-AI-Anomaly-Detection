---

🎯 Motivation

Traditional RAN monitoring relies heavily on static thresholds and manual troubleshooting.

This project explores how modern observability techniques and machine learning can be applied to mobile networks by creating a complete monitoring pipeline inspired by real O-RAN management systems.

The goal is not to simulate KPIs but to build a realistic workflow where metrics are generated by a functioning radio stack, transported through management interfaces, stored in a time-series database, and analyzed automatically.

🧠 Machine Learning Pipeline

Isolation Forest

Detects sudden abnormal events such as:

- BLER spikes
- Unexpected CQI degradation
- PRB saturation
- Handover failures

LSTM Autoencoder

Detects gradual performance degradation such as:

- Slowly increasing congestion
- Long-term radio quality deterioration
- Capacity exhaustion trends

Using both approaches allows the system to identify both instantaneous faults and slow network drift.
